# BPMS 자동화 (로컬 PC 버전)

Salesforce BPMS의 4개 보고서뷰를 자동으로 다운로드하는 Python 스크립트.

## 📋 폴더 구성

```
automation/
├── bpms_download.py    # 메인 스크립트
├── requirements.txt    # Python 의존성
├── install.bat         # 원클릭 설치 (Windows)
├── run.bat             # 원클릭 실행 (Windows)
├── README.md           # 이 파일
├── downloads/          # 다운로드된 엑셀 파일들
└── session_state.json  # 로그인 세션 (자동 생성)
```

## 🚀 처음 설치 (1회만)

1. **이 폴더 통째로** 영업현황 자동화 폴더로 복사 (이미 됨)
2. `install.bat` 더블클릭
3. 5~10분 정도 기다림 (Playwright + Chromium 다운로드)
4. "✅ 모든 설치 완료!" 메시지 확인

## 📥 사용 방법

1. **`run.bat` 더블클릭**
2. 브라우저가 자동으로 열림 → Salesforce 로그인 페이지
3. **직접 ID/비밀번호 입력해서 로그인** (자동 로그인 X — 이번에는 수동)
4. 메인 화면이 보이면 콘솔로 돌아가서 **Enter** 누름
5. 자동으로 4개 보고서뷰 차례로 열고 엑셀 다운로드
6. `downloads/` 폴더에 저장됨

## 📂 다운로드 파일 예시

```
downloads/
├── 영업정보_20260520_141023_BPMS_영업현황_자동화_영업정보.xlsx
├── 충전기정보_20260520_141145_BPMS_영업현황_자동화_충전기정보.xlsx
├── 전암검외_20260520_141310_...xlsx
└── 세금계산서_20260520_141445_...xlsx
```

각 파일명에:
- 보고서 이름
- 다운로드 시각 (YYYYMMDD_HHMMSS)
- 원본 파일명

## ⚠ 주의사항

- 첫 로그인 시 SMS/OTP 인증이 있다면 직접 처리
- 회사 정책상 자동 로그인이 금지될 수 있음 → 매번 직접 로그인
- 보고서 행이 매우 많으면 다운로드 시간이 오래 걸림 (~3분)
- 브라우저 창을 직접 닫지 말 것 (스크립트가 정리 후 자동 종료)

## 🔧 자주 묻는 질문

### Q. "메뉴 버튼을 찾지 못했습니다"
A. Salesforce UI가 약간 변경됐을 수 있어요. 스크린샷과 함께 문의해 주세요.

### Q. "다운로드가 안 끝나요"
A. 보고서 행이 많으면 시간이 걸립니다. 콘솔에 "다운로드 중..." 표시되어 있으면 계속 기다려 주세요.

### Q. 다음번에 자동 로그인 가능한가요?
A. `session_state.json` 파일이 만들어지면 Salesforce 세션을 기억합니다. 단, Salesforce는 보통 8시간~1일 후 세션이 만료되어 다시 로그인이 필요합니다.

### Q. 로그인이 자동으로 되도록 만들고 싶어요
A. 다음 단계로 — 회사 정책에 따라 자격증명을 안전하게 저장하는 방법으로 작업할 수 있어요.
